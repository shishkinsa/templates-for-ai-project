# Диаграммы архитектуры

Каталог для C4-моделей (LikeC4), схем данных и поведенческих диаграмм.

## Текущее состояние шаблона

| Путь | Статус |
|------|--------|
| `context/01-model.c4` | Контекст и контейнеры (3 узла) |
| `containers/01-views.c4` | Представления контейнерной диаграммы |
| `containers/cnt_sp_web/01-model.c4` | Заготовка Web SPA |
| `containers/cnt_sp_webapi/01-model.c4` | Заготовка WebAPI |
| `containers/cnt_sp_db/01-model.c4` | Заготовка PostgreSQL |
| `data/cnt_sp_db/20-database-structure.md` | Описание схемы БД |

Остальные каталоги (`components/`, `behavior/`, `infrastructure/`, `security/`, …) — placeholder'ы. Добавляйте по мере роста проекта.

## Структура папок

```text
docs/architecture/diagram/
├── README.md
├── context/               # 01–09: контекст системы
├── containers/            # 01–09: контейнеры (cnt_*)
├── components/            # 10–19: компоненты внутри контейнеров
├── data/                  # 20–29: схемы БД, ER
├── infrastructure/        # 30–39: деплой, сети
├── behavior/              # 40–49: последовательности, сценарии
├── business/              # 50–59: бизнес-процессы
├── security/              # 60–69: аутентификация, угрозы
├── integration/           # 70–79: внешние системы
└── legacy/                # 80–99: устаревшие диаграммы
```

### Просмотр LikeC4

```powershell
# Из корня репозитория
npx likec4 validate docs/architecture/diagram
npx likec4 preview docs/architecture/diagram
```

Экспорт PNG — по необходимости (`npx likec4 export`). Готового CI-шага для диаграмм в шаблоне нет.

## Связь с требованиями

В начале файла диаграммы указывайте:

```likec4
// Связанные требования: FR-001
// ADR: 0001, 0002, 0003
```

При изменении контейнеров обновляйте:

1. LikeC4-модели в этом каталоге
2. `docs/ai/project-context.md`
3. ADR при смене технологий
